## ⚙️ Enable the advanced widgets (stats, languages, achievements, habits, calendar)

Five sections in this README — **GitHub Analytics** (stats + languages), **GitHub Achievements**, **Coding Habits**, and **Animated Contribution Calendar** — are placeholders until you run a one-time GitHub Action setup. They're all generated by the same `lowlighter/metrics` Action, which pulls data GitHub doesn't expose publicly and also sidesteps the outages on the old public stats/trophy services.

1. Go to **github.com/settings/tokens** → generate a **classic** personal access token, no scopes required (unless your commits are on private repos, in which case add `repo`).
2. In your **`scout6774-66667/scout6774-66667`** profile repo, go to **Settings → Secrets and variables → Actions** and add a new secret named `METRICS_TOKEN` with that token as the value.
3. Create a file at `.github/workflows/metrics.yml` in that repo with the contents from `metrics-workflow.yml` (provided alongside this README).
4. Commit it, then run the workflow once manually from the **Actions** tab (or wait for the daily schedule). It will auto-commit five SVGs into a `github-metrics/` folder in your repo: `general.svg`, `languages.svg`, `achievements.svg`, `habits.svg`, and `isocalendar.svg`.
5. All the `<img>` tags already in this README point to those exact file paths — nothing else to edit once step 4 finishes.

## 🔧 Fixing broken widgets

- **Stats, Top Languages, and Trophies** were previously served by `github-readme-stats.vercel.app` and `github-profile-trophy.vercel.app` — both have had well-documented, widespread outages since late 2025 due to rate limiting on their shared free instance. This README now generates those as `general.svg`, `languages.svg`, and `achievements.svg` via your own GitHub Action instead (see the setup section above), which avoids the problem entirely.
- **Streak stats** (`streak-stats.demolab.com`) has stayed reliable, so it's left as a hotlinked badge.
- If any of the self-hosted `github-metrics/*.svg` images show as broken links, it almost always means the Action hasn't run yet, or `METRICS_TOKEN` wasn't set correctly — check the **Actions** tab in your profile repo for a red ❌ and click in to see the error.
- Always double check the `username=` value matches your exact GitHub handle — a typo silently returns an empty card instead of an error.
